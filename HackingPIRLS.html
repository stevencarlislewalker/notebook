Hacking the deviance function

Since devfun0 changes (or presumably more precisely, since the environment of devfun0 changes), perhaps we could hack it to work by forcing it to reset back to its original definition every time it gets called. To do this, I create a function that resets the deviance function before each evaluation:

fn <- function(pars) {
    devfun <- glmer(form, weights = eggs, data = ss, family = binomial, nAGQ = 1L, 
        devFunOnly = TRUE)
    dev <- try(devfun(pars), silent = TRUE)

    if (inherits(dev, "try-error")) 
        return(Inf) else return(dev)
}

Working with the hacked deviance function

The simplest thing to do is to pass fn to optim,
