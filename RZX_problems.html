Illustrating trouble with RZX in development lme4

What is RZX?

A development version of the lme4 package iteratively uses block Cholesky decompositions for fitting generalized linear mixed models. In particular, the block matrix that is Cholesky decomposed is defined by, \[ \begin{bmatrix} \mathbf{P}\left(\Lambda_\theta^\top \mathbf{Z}^\top \mathbf{W}^{-1} \mathbf{Z}\Lambda_\theta+ \mathbf{I}_q\right) \mathbf{P}^\top & \mathbf{P}\Lambda_\theta^\top \mathbf{Z}^\top \mathbf{W}^{-1} \mathbf{X}\\ \mathbf{X}^\top \mathbf{W}^{-1} \mathbf{Z}\Lambda_\theta \mathbf{P}^\top & \mathbf{X}^\top \mathbf{W}^{-1} \mathbf{X} \end{bmatrix} \] where \( \mathbf{Z} \) and \( \mathbf{X} \) are the random- and fixed-effects design matrices, \( \mathbf{W} \) is a diagonal weight (i.e. inverse variance) matrix, \( \Lambda_\theta \) is a triangular matrix determining the covariance matrix of the random-effects, and \( \mathbf{P} \) is a permutation matrix (which is not explicitly calculated). The Cholesky decomposition of this matrix takes the form, \[ \begin{bmatrix} \mathbf{L}_\theta & \mathbf{0} \\ \mathbf{R}^\top_{ZX} & \mathbf{R}^\top_{X} \end{bmatrix} \begin{bmatrix} \mathbf{L}^\top_\theta & \mathbf{R}_{ZX} \\ \mathbf{0} & \mathbf{R}_{X} \end{bmatrix} \] The three blocks of this decomposition are computed in the following manner. First, \( \mathbf{L}_\theta \) is the Cholesky factor of the first block, \[ \mathbf{L}_\theta \mathbf{L}^\top_\theta = \mathbf{P}\left(\Lambda_\theta^\top \mathbf{Z}^\top \mathbf{W}^{-1} \mathbf{Z}\Lambda_\theta+ \mathbf{I}_q\right) \mathbf{P}^\top \] With this Cholesky factor, \( \mathbf{R}_{ZX} \) can be calculated as the solution to the following linear equation, \[ \mathbf{L}_\theta \mathbf{R}_{ZX} = \mathbf{P}\Lambda_\theta^\top \mathbf{Z}^\top \mathbf{W}^{-1} \mathbf{X} \] Finally, the matrix \( \mathbf{R}_{X} \) can be obtained via the downdated Cholesky decomposition, \[ \mathbf{R}^\top_{X} \mathbf{R}_{X} = \mathbf{X}^\top \mathbf{W}^{-1} \mathbf{X} - \mathbf{R}^\top_{ZX} \mathbf{R}_{ZX} \]

For some reason, the calculation of \( \mathbf{R}_{ZX} \) is not being made properly in development lme4 for some examples. Here we describe one such example.

Modifications to lme4 for debugging purposes

The following modifications are incorporated into my branch of lme4 on github. To print out the values of the RZX matrices computed by lme4, we modified the merPredD::updateDecomp function in the predModule.cpp file:

void merPredD::updateDecomp() { // update L, RZX and RX
  updateL();
  d_RZX         = d_LamtUt * d_V;
  Rcpp::Rcout << "RZX:\n" << d_RZX << std::endl;

The modification is the final line in this snipet, which prints out the RZX matrix everytime it is computed.

We also modify the glmer function in lmer.R to calculate and print out RZX 'by hand'. First, we create two lists of variables (pp_byhand and resp_byhand), which contain the ingredients necessary to make the first calculation of RZX, and then we compute and print out RZX itself,

rho <- as.environment(list(verbose = verbose, tolPwrss = tolPwrss))
parent.env(rho) <- parent.frame()

pp_byhand <- c(reTrms[c("Zt", "theta", "Lambdat", "Lind")], n = nrow(X), list(X = X))
resp_byhand <- mkRespMod(fr, family = family)
RZXhand <- RZXbyhand(pp = pp_byhand, resp = resp_byhand, includeH = FALSE)
